DEPLOYMENT GUIDE: Admin Features & Enhanced Schema
====================================================

This guide covers deploying the new admin management features to the Hestia server.

PHASE 1: DATABASE SCHEMA MIGRATION
===================================
1. SSH or use Hestia File Manager to access the database:
   - Hestia Control Panel → Databases → neovtrack_db → phpMyAdmin
   - Or use command line: mysql -u [user] -p [database]

2. Run the migration script (database/migration_admin_features.sql):
   - Copy entire content of migration_admin_features.sql
   - In phpMyAdmin: Click "SQL" tab, paste content, click "Go"
   - Or via CLI: mysql -u [user] -p [database] < migration_admin_features.sql

3. Verify tables created:
   - visitorcar table exists with fields: visitorid, name, phone, ic_passport, model, platenum, sticker, sticker_status, created_at, updated_at
   - contractorcar table exists with fields: contractorid, name, phone, ic_passport, company_name, model, platenum, sticker, sticker_status, created_at, updated_at
   - admin_users table exists
   - admin_action_logs table exists

4. Verify existing tables updated:
   - staffcar: check for created_at, updated_at, sticker_status columns
   - studentcar: check for created_at, updated_at, sticker_status columns
   - admin table: check for role, status, created_at, updated_at columns
   - user table: check for created_at, updated_at columns

PHASE 2: DEPLOY API ENDPOINTS
==============================
Deploy the following files to the webroot (public_html) via Hestia File Manager or SFTP:

1. admin_management_api.php
   - Location: Upload to public_html (same directory as admin.php, user.php)
   - Permissions: 644
   - Purpose: Add/manage admins and users (superadmin only)
   - Endpoint: POST admin_management_api.php?action=add_admin or action=add_user
   - GET admin_management_api.php?action=list_admins or action=list_users

2. vehicle_stats_api.php
   - Location: Upload to public_html
   - Permissions: 644
   - Purpose: Get vehicle statistics and filtered vehicle lists
   - Endpoint: GET vehicle_stats_api.php?action=get_stats
   - GET vehicle_stats_api.php?action=get_vehicles_by_type&type=staff&limit=50&offset=0

3. sticker_management_api.php
   - Location: Upload to public_html
   - Permissions: 644
   - Purpose: Remove/restore vehicle stickers
   - Endpoint: POST sticker_management_api.php?action=remove_sticker
   - POST sticker_management_api.php?action=restore_sticker

PHASE 3: DEPLOY WEB INTERFACE
=============================
Deploy the following file to the webroot:

1. superadmin.php
   - Location: Upload to public_html (same directory as admin.php)
   - Permissions: 644
   - Purpose: Superadmin dashboard with vehicle statistics and admin management
   - Access: http://neovtrack.uitm.edu.my/superadmin.php
   - Authorization: Only admins with userid <= 10 (original admins)

PHASE 4: UPDATE EXISTING FILES (OPTIONAL)
==========================================
Update the following files to add links to superadmin dashboard:

1. admin.php
   - Add navigation link: <a href="superadmin.php">Superadmin Dashboard</a>
   - Only display for superadmin (userid <= 10)

2. menus.php
   - Add superadmin menu item for quick access

PHASE 5: TESTING
================
After deployment, test each endpoint:

1. Test vehicle statistics API:
   curl -X GET https://neovtrack.uitm.edu.my/vehicle_stats_api.php?action=get_stats
   Expected: JSON with stats object containing staff, student, visitor, contractor, total, removed counts

2. Test vehicle list by type:
   curl -X GET https://neovtrack.uitm.edu.my/vehicle_stats_api.php?action=get_vehicles_by_type&type=staff&limit=10&offset=0
   Expected: JSON with vehicles array

3. Test superadmin dashboard:
   Visit: https://neovtrack.uitm.edu.my/superadmin.php
   Expected: Dashboard loads with vehicle statistics and admin management forms

4. Test add admin (requires valid admin session):
   curl -X POST https://neovtrack.uitm.edu.my/admin_management_api.php \
     -d "action=add_admin&email=newadmin@example.com&name=New Admin&password=securepass"
   Expected: JSON response with success=true and new admin_id

5. Test sticker removal:
   curl -X POST https://neovtrack.uitm.edu.my/sticker_management_api.php \
     -d "action=remove_sticker&table=staffcar&record_id=1"
   Expected: JSON response with success=true

DEPLOYMENT VIA HESTIA FILE MANAGER
===================================
1. Log in to Hestia Control Panel (http://hestia.uitm.edu.my:8083)
2. Click Web → neovtrack.uitm.edu.my
3. Click File Manager
4. Navigate to public_html
5. Click Upload and select files:
   - admin_management_api.php
   - vehicle_stats_api.php
   - sticker_management_api.php
   - superadmin.php
6. Set permissions to 644 for all PHP files
7. Test each file by visiting the URL

DEPLOYMENT VIA SFTP
===================
1. Open SFTP client (WinSCP, FileZilla, etc.)
2. Connect to hestia.uitm.edu.my:22 with credentials
3. Navigate to public_html directory
4. Upload the following files:
   - admin_management_api.php
   - vehicle_stats_api.php
   - sticker_management_api.php
   - superadmin.php
5. Set permissions to 644 (right-click → Properties → Permissions)

TROUBLESHOOTING
===============
- If 404 error: Check file placement in public_html, verify permissions
- If 403 error: Check file permissions and owner/group
- If admin_management_api.php returns 403: Ensure user is logged in as admin with userid <= 10
- If tables don't exist: Run migration script again or check MySQL error logs
- If API returns error: Check database connection in connect.php
- If forms don't submit: Check browser console for JavaScript errors

ROLLBACK (IF NEEDED)
====================
1. To remove new tables (caution: data loss):
   - DROP TABLE admin_action_logs;
   - DROP TABLE admin_users;
   - DROP TABLE visitorcar;
   - DROP TABLE contractorcar;

2. To revert column changes on existing tables:
   - ALTER TABLE staffcar DROP COLUMN created_at, DROP COLUMN updated_at, DROP COLUMN sticker_status;
   - ALTER TABLE studentcar DROP COLUMN created_at, DROP COLUMN updated_at, DROP COLUMN sticker_status;
   - Similar for admin and user tables

3. Delete API files:
   - admin_management_api.php
   - vehicle_stats_api.php
   - sticker_management_api.php
   - superadmin.php
