MANUAL STARTUP STEPS (if scripts don't work)
================================================================================

1. START MYSQL
   ============
   Open PowerShell as Admin or use XAMPP Control Panel:
   
   Via Command Line:
   - cd C:\xampp\mysql\bin
   - mysqld --port=3306 --datadir="C:\xampp\mysql\data" --default-storage-engine=InnoDB
   
   Via GUI:
   - Start XAMPP Control Panel (C:\xampp\xampp-control.exe)
   - Click "Start" next to MySQL
   
   Verify MySQL is running:
   - cd C:\xampp\mysql\bin
   - mysql -u root -e "SELECT 1;"
   - Should show: | 1 |

   DATABASE INITIALIZATION (first time only):
   If neovtrack_db doesn't exist yet:
   - cd C:\xampp\mysql\bin
   - mysql -u root < "C:\Users\User.J1-ALPHA-PENS\pbsys\database\neovtrack_db.sql"
   
   Or via GUI (phpMyAdmin):
   - Start XAMPP Control Panel, click "Admin" for MySQL (opens phpMyAdmin)
   - Import C:\Users\User.J1-ALPHA-PENS\pbsys\database\neovtrack_db.sql

2. START PHP BUILT-IN SERVER
   ===========================
   Open a new terminal and:
   cd C:\Users\User.J1-ALPHA-PENS\pbsys
   C:\xampp\php\php.exe -S localhost:8000
   
   Verify:
   - Open browser: http://localhost:8000
   - Should NOT 404 on root (shows directory listing or connect.php)
   - Backend API working: http://localhost:8000/api/

3. LAUNCH ANDROID EMULATOR
   ========================
   In a new terminal:
   flutter emulators --launch Pixel_API_36
   
   Wait for emulator to fully boot (1-2 minutes).
   Can also use: adb devices (should list the emulator as device)

4. VERIFY DATABASE CONNECTION
   ===========================
   In yet another terminal:
   cd C:\Users\User.J1-ALPHA-PENS\pbsys
   C:\xampp\php\php.exe -r "
   include 'includes/connect.php';
   echo 'Database connected: ' . ($conn ? 'OK' : 'FAILED') . PHP_EOL;
   "

5. LAUNCH FLUTTER APP

   flutter run -d android --dart-define=API_BASE_URL=http://localhost:8000

================================================================================
TROUBLESHOOTING
================================================================================

MySQL won't start:
  - Already running? Check taskbar or: netstat -ano | findstr 3306
  - Port conflict? Change to different port in startup commands
  - Data dir locked? Close any running mysqld processes

PHP server won't start:
  - Port 8000 in use? lsof -i :8000 or netstat -ano | findstr 8000
  - Use different port: php.exe -S localhost:9000

Emulator won't launch:
  - Check: flutter doctor -v
  - Verify Android SDK: adb devices
  - May need to close and retry: emulator -avd Pixel_API_36 -wipe-data

Flutter app can't connect to API:
  - Verify PHP server is running and responding: curl http://localhost:8000/api/
  - Check API_BASE_URL in Flutter: --dart-define=API_BASE_URL=http://localhost:8000
  - Within emulator, localhost:8000 might need to be 10.0.2.2:8000 (Android emulator's gateway)
  - If using network IP for CI/runner: change to --dart-define=API_BASE_URL=http://<YOUR_IP>:8000

================================================================================
LOGS
================================================================================

PHP server output: printed to terminal window
MySQL output: check startup terminal
Emulator output: check Android Studio logcat or emulator window
Flutter app: check terminal where "flutter run" was launched

For persistent logs:
  PHP server redirect: C:\xampp\php\php.exe -S localhost:8000 > php-server.log 2>&1
  Flutter logs: flutter logs

================================================================================
WHEN DONE: CLEANUP
================================================================================

Stop Flutter app: Press Q in the terminal, or Ctrl+C
Stop PHP server: Ctrl+C in terminal
Stop Emulator: Close emulator window (or `adb emu kill`)
Stop MySQL:
  - Via XAMPP: Control Panel > Stop MySQL
  - Via CLI: taskkill /IM mysqld.exe /F (Windows)

================================================================================
DEPLOY WITH LOCAL RUNNER (CI/RUNNER_NOTES)
================================================================================

When deploying via J1-OMEGA-30 runner (UiTM LAN):
  1. Replace --dart-define=API_BASE_URL=http://localhost:8000 with runner's local IP:
     flutter run -d android --dart-define=API_BASE_URL=http://10.0.X.X:8000
  2. Or use network hostname if resolvable on LAN
  3. Ensure PHP server listens on 0.0.0.0, not just localhost:
     php.exe -S 0.0.0.0:8000
